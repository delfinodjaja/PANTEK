# PANTEK | Predictive Analytics Network

## Overview
**PANTEK** (also known as **NeuroChain**) is a next-generation Global Supply Intelligence platform. It leverages a "Digital Twin" architecture combined with multi-layered AI simulation (Gemini) to predict, analyze, and mitigate global supply chain risks.

The system acts as a world modeler, allowing users to inject "operational shocks" (droughts, strikes, cyber-attacks) and observe their propagation through the network in real-time.

---

## 🛠 Key Features

- **Digital Twin Visualization**: Real-time interactive maps showing suppliers, warehouses, and global shipping routes.
- **AI Scenario Lab**: A sandbox to simulate complex risks using natural language prompts.
- **Cascade Intelligence**: Automatic detection of second and third-order ripple effects across the supply chain.
- **Auto-Heal Optimization**: AI-driven solver that suggests and applies mitigation strategies to resolve bottlenecks.
- **Live Monitoring**: Real-time health metrics of all network nodes (Centrality, Flow, and Stress).
- **Analysis Engine**: Deep-dive probabilistic assessments with explainable AI reasoning.

## 📖 Tutorial: Navigating the System

### 1. The Command Center (Overview & Live Map)
- **Dashboard**: Start at the **Overview** to see your "Global Risk Index" and active disruptions.
- **Live Map**: Navigate to the **Live Map** to see the heartbeat of your supply chain. 
    - *Hover* over nodes to see throughput and inventory.
    - *Filter* by asset type (Air, Sea, Land) to isolate specific logistics layers.

### 2. Injecting a Shock (Scenario Lab)
The **Scenario Lab** is where the predictive power of PANTEK comes alive.
1. **Choose a Preset**: Try "Panama Drought" or "Cyber Lockdown" to see immediate propagation.
2. **Custom Prompts**: In the **Simulation Propagator**, type a scenario like: *"Simulate a 48-hour rail strike in Germany affecting land routes between Berlin and Munich."*
3. **Neural Loop**: Watch the indicator as LLM 1 models the world and LLM 2 critiques the logic for high-fidelity convergence.

### 3. Timeline Scrubber (Dynamic Propagation)
Once a simulation runs:
- Use the **Timeline Scrubber** at the bottom of the map to move from **Day 1 to Day 30**.
- Notice how nodes turn red (Stressed) or flash (Critical) as the disruption propagates over time.
- The **Heatmap** toggle allows you to visualize high-risk clusters across the globe.

### 4. Mitigating Risks (Auto-Heal)
1. After a simulation, a list of **Mitigation Strategies** will appear.
2. Manually toggle strategies (e.g., "Air Freight Bridge") to see the "Mitigated" vs "Baseline" impact on the map.
3. **Auto-Heal Button**: Click this to let the AI Optimization Solver automatically pick the best combination of strategies to resolve current bottlenecks.

### 5. Deep Analysis
Go to the **Analysis Engine** for a structured breakdown:
- **Financial Impact**: Estimated losses.
- **Bottlenecks**: Specific facilities at risk.
- **AI Recommendations**: Tactical and Strategic advice generated by the AI Intelligence Assessment.

## ⚙️ Technical Configuration

### Prerequisites
- **Gemini API Key**: Required for the simulation and analysis engines.
- **Node.js**: Standard environment for the Express/Vite backend.

### Environment Variables
Create a `.env` file based on `.env.example`:
```env
GEMINI_API_KEY=your_key_here
APP_URL=https://your-app.netlify.app
```

### Installation
1. Install dependencies: `npm install`
2. Start the development server: `npm run dev`
3. Access the dashboard at your deployed URL (e.g. Netlify).

## 📊 System Concepts

- **Centrality**: Measures how critical a node is to the entire network flow.
- **Propagation**: The movement of stress from a failed node to its connected partners.
- **Stochastic Analysis**: Using probabilistic models to account for uncertainty in global events.
